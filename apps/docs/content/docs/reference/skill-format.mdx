Each skill is a **directory** containing a `SKILL.md` file with YAML frontmatter metadata, and optionally additional supporting files. This page documents the complete format.

## Directory Structure

Each skill lives in its own directory:

```
my-skill/
  SKILL.md            # Required — main skill content
  references/          # Optional — supporting documents
  scripts/             # Optional — helper scripts
  assets/              # Optional — images, data files, etc.
```

## SKILL.md Structure

The `SKILL.md` file has two parts separated by `---` delimiters:

```markdown
---
name: "Skill Name"
description: "A short description of what this skill covers."
tags: ["tag1", "tag2"]
version: "1.0"
author: "Author Name"
created: "2026-01-15"
updated: "2026-03-20"
---

# Heading

Your markdown content goes here. This is the skill body that
the AI assistant will read when it retrieves this skill.
```

## Frontmatter Fields

### Base Fields

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | Yes | Human-readable display name |
| `description` | `string` | Yes | Short description (shown in skill cards and search results) |
| `tags` | `string[]` | No | Tags for categorization and search |
| `version` | `string` | No | Version identifier (e.g. `"1.0"`, `"2.3"`) |
| `author` | `string` | No | Author name |
| `created` | `string` | No | ISO date of creation (e.g. `"2026-01-15"`) |
| `updated` | `string` | No | ISO date of last modification |

### Claude Code Fields

These optional fields control how Claude Code handles the skill:

| Field | Type | Description |
|---|---|---|
| `allowed-tools` | `string` | Pre-approved tools, e.g. `"Read, Grep, Glob, Bash(npm test *)"` |
| `disable-model-invocation` | `boolean` | If `true`, only users can invoke via `/name` (Claude cannot auto-trigger) |
| `user-invocable` | `boolean` | If `false`, acts as background knowledge only (not invocable via `/`) |
| `context` | `string` | Execution context — `"fork"` runs in an isolated subagent |
| `agent` | `string` | Agent type when forked, e.g. `"Explore"`, `"Plan"` |
| `model` | `string` | Model override, e.g. `"haiku"`, `"sonnet"`, `"opus"` |
| `argument-hint` | `string` | Hint shown after `/name`, e.g. `"<file> [--verbose]"` |

## Directory Name Convention

The directory name becomes the **skill ID**:

| Directory | Skill ID |
|---|---|
| `typescript-best-practices/` | `typescript-best-practices` |
| `react-patterns/` | `react-patterns` |
| `api-design/` | `api-design` |

The skill ID is used in all MCP tool calls (`skills__get`, etc.) and must be unique.

## Body Content

The body can contain any valid markdown:

- Headings (`##`, `###`, etc.)
- Lists (ordered and unordered)
- Code blocks with syntax highlighting
- Tables
- Links
- Bold, italic, strikethrough
- Blockquotes

## Complete Example

```markdown
---
name: "React Component Patterns"
description: "Common React component patterns including composition, render props, and hooks."
tags: ["react", "components", "patterns", "frontend"]
version: "2.0"
author: "Jane Doe"
created: "2026-01-10"
updated: "2026-03-18"
allowed-tools: "Read, Grep, Glob"
context: fork
agent: Explore
---

## Composition Pattern

Prefer composition over inheritance. Use children or render props
to make components flexible:

\`\`\`tsx
function Card({ children }: { children: React.ReactNode }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Title</h2>
      <p>Content goes here.</p>
    </Card>
  );
}
\`\`\`

## Custom Hooks

Extract reusable logic into custom hooks:

\`\`\`tsx
function useLocalStorage<T>(key: string, initialValue: T) {
  const [value, setValue] = useState<T>(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue] as const;
}
\`\`\`

## Key Principles

- Keep components small and focused
- Lift state up only when necessary
- Use TypeScript for prop type safety
- Prefer controlled components for forms
```

## Storage Location

Skills are stored in the configured skills directory:

```
~/.skill-management/skills/
  typescript-best-practices/
    SKILL.md
  react-patterns/
    SKILL.md
    references/
      hooks-api.md
  api-design/
    SKILL.md
```

The directory path can be changed in the [configuration](/docs/reference/config) via the `skills.directory` field.

Skills are automatically symlinked to `~/.claude/skills/` for Claude Code integration.
