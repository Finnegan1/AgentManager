Full reference for `~/.skill-management/config.json`.

## Top-Level Fields

| Field | Type | Required | Description |
|---|---|---|---|
| `version` | `"1.0"` | Yes | Config schema version |
| `gateway` | `object` | Yes | Gateway runtime settings |
| `servers` | `Record<string, ServerConfig>` | Yes | Downstream server definitions |
| `skills` | `object` | Yes | Skills directory configuration |

## `gateway`

| Field | Type | Default | Description |
|---|---|---|---|
| `autoStart` | `boolean` | `false` | Whether to auto-start the gateway when the desktop app launches |
| `port` | `number` | `24842` | Port for the HTTP daemon (1–65535) |

## `servers`

A map of server key → server configuration. The key is used as the namespace prefix for all proxied capabilities.

**Server key rules:**
- Must be unique across all servers
- Used as the namespace prefix (e.g. `db__tool_name`)
- Should be short and descriptive (e.g. `db`, `ai-tools`, `web-search`)

### Server Configuration

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | Yes | Human-readable display name |
| `enabled` | `boolean` | Yes | Whether the server is active |
| `scope` | `"global" \| "project"` | No | Server scope. Default: `"global"` |
| `transport` | `TransportConfig` | Yes | Connection configuration |

### Scope

| Value | Behavior |
|---|---|
| `"global"` (default) | Started once, shared across all client sessions. Use for APIs, databases, web services. |
| `"project"` | Started per project root with the project's `cwd`. Sessions with the same root share connections. Use for filesystem, linters, Git, build tools. |

### Transport: stdio

| Field | Type | Required | Description |
|---|---|---|---|
| `type` | `"stdio"` | Yes | Transport type identifier |
| `command` | `string` | Yes | Executable to launch |
| `args` | `string[]` | No | Command-line arguments |
| `env` | `Record<string, string>` | No | Environment variables |
| `cwd` | `string` | No | Working directory for the process. For `scope: "project"` servers, this is automatically overridden to the project root. |

```json
{
  "type": "stdio",
  "command": "bun",
  "args": ["/path/to/server.ts", "--verbose"],
  "env": { "API_KEY": "sk-123", "DEBUG": "true" },
  "cwd": "/path/to/project"
}
```

### Transport: SSE

| Field | Type | Required | Description |
|---|---|---|---|
| `type` | `"sse"` | Yes | Transport type identifier |
| `url` | `string` | Yes | HTTP endpoint URL |
| `headers` | `Record<string, string>` | No | Custom HTTP headers |

```json
{
  "type": "sse",
  "url": "https://example.com/mcp",
  "headers": { "Authorization": "Bearer token123" }
}
```

### Transport: Streamable HTTP

| Field | Type | Required | Description |
|---|---|---|---|
| `type` | `"streamable-http"` | Yes | Transport type identifier |
| `url` | `string` | Yes | HTTP endpoint URL |
| `headers` | `Record<string, string>` | No | Custom HTTP headers |

```json
{
  "type": "streamable-http",
  "url": "https://example.com/mcp",
  "headers": { "Authorization": "Bearer token123" }
}
```

## `skills`

| Field | Type | Default | Description |
|---|---|---|---|
| `directory` | `string` | `"~/.skill-management/skills"` | Path to the skills directory |

## `status.json` (Runtime Status)

The gateway writes its runtime status to `~/.skill-management/status.json` every 5 seconds. This is not user-editable — it's generated by the running gateway process.

| Field | Type | Description |
|---|---|---|
| `pid` | `number` | Process ID of the gateway |
| `startedAt` | `string` | ISO timestamp of when the gateway started |
| `port` | `number` | Port the HTTP daemon is listening on |
| `url` | `string` | Full URL for MCP clients to connect to |
| `activeSessions` | `number` | Number of currently connected client sessions |
| `servers` | `Record<string, ServerStatus>` | Status of each downstream connection |

Each server status entry:

| Field | Type | Description |
|---|---|---|
| `connected` | `boolean` | Whether the server is connected |
| `toolCount` | `number` | Number of tools exposed |
| `resourceCount` | `number` | Number of resources exposed |
| `promptCount` | `number` | Number of prompts exposed |
| `error` | `string?` | Error message if connection failed |

## Complete Example

```json
{
  "version": "1.0",
  "gateway": {
    "autoStart": false,
    "port": 24842
  },
  "servers": {
    "database": {
      "name": "Database Tools",
      "enabled": true,
      "scope": "global",
      "transport": {
        "type": "stdio",
        "command": "bun",
        "args": ["/path/to/db-mcp/index.ts"],
        "env": {
          "DATABASE_URL": "postgresql://localhost:5432/mydb"
        }
      }
    },
    "remote-api": {
      "name": "Remote API",
      "enabled": true,
      "scope": "global",
      "transport": {
        "type": "sse",
        "url": "https://api.example.com/mcp",
        "headers": {
          "Authorization": "Bearer sk-abc123"
        }
      }
    },
    "filesystem": {
      "name": "Filesystem",
      "enabled": true,
      "scope": "project",
      "transport": {
        "type": "stdio",
        "command": "npx",
        "args": ["@mcp/filesystem", "."]
      }
    }
  },
  "skills": {
    "directory": "~/.skill-management/skills"
  }
}
```
