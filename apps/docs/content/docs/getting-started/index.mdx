## Prerequisites

Before you begin, make sure you have the following installed:

| Requirement | Version | Purpose |
|---|---|---|
| [Bun](https://bun.sh) | 1.3.11+ | Package manager and runtime |
| [Node.js](https://nodejs.org) | 18+ | Required by some tooling |
| [Rust](https://rustup.rs) | Latest stable | Tauri desktop app backend |

You'll also need the Tauri CLI prerequisites for your platform. See the [Tauri prerequisites guide](https://v2.tauri.app/start/prerequisites/) for details.

## Installation

Clone the repository and install dependencies:

```bash
git clone https://github.com/Finnegan1/skills-mcp.git
cd skills-mcp/agent-manager
bun install
```

## First Launch

Start the desktop app in development mode:

```bash
cd apps/agent-manager
bun run tauri dev
```

This will:
1. Start the Vite dev server for the React frontend
2. Compile and launch the Tauri (Rust) backend
3. Open the Agent Manager desktop application

On first launch, the app creates the configuration directory at `~/.agent-manager/` with a default `config.json` and an empty `skills/` folder.

## Starting the MCP Gateway

The MCP gateway runs as an HTTP daemon — a long-running background process that multiple AI clients can connect to simultaneously. Start it with:

```bash
cd apps/mcp-gateway
bun run dev:server
```

Or for production (requires building first with `bun run build`):

```bash
cd apps/mcp-gateway
bun run start
```

The gateway starts listening on `http://127.0.0.1:24842/mcp` by default. You can specify a custom port:

```bash
bun run dev:server -- --port 3000
```

<Callout>
  The gateway must be running before your AI client can connect. Unlike stdio-based servers, the gateway is **not** automatically started by the AI client — it runs as an independent daemon process.
</Callout>

## Connecting Your AI Client

Configure your AI client to connect to the running gateway. For Claude Code, add this to your `.mcp.json`:

```json
{
  "mcpServers": {
    "skill-gateway": {
      "type": "http",
      "url": "http://127.0.0.1:24842/mcp"
    }
  }
}
```

<Callout>
  The desktop app and the MCP gateway communicate through shared files on disk — they don't need a direct network connection. The gateway writes its status to `~/.agent-manager/status.json`, and the desktop app polls it every 5 seconds.
</Callout>
