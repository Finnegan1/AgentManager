## Component Responsibilities

### MCP Gateway HTTP Daemon (`apps/mcp-gateway/`)

A Node.js/Bun HTTP daemon that runs on `127.0.0.1:24842`:

- **Accepts HTTP connections** from multiple AI clients simultaneously
- **Manages sessions** — each client gets its own `Server` + `StreamableHTTPServerTransport` pair
- **Serves skills** from `~/.agent-manager/skills/` via native tools
- **Proxies downstream servers** — connects to all enabled servers, namespaces their capabilities, and forwards requests
- **Splits by scope** — global-scope servers are shared; project-scope servers are managed per project root
- **Watches configuration** — picks up changes to `config.json` without restart
- **Writes status** — periodically writes its PID, port, URL, active sessions, and connection states to `status.json`

Key internal components:

| Component | Responsibility |
|---|---|
| **HTTP Server** | Routes incoming requests by session ID |
| **SessionManager** | Creates/removes session pairs, broadcasts notifications, cleans up inactive sessions (30min timeout) |
| **Global ProxyManager** | Manages downstream connections for `scope: "global"` servers |
| **ProjectConnectionPool** | Manages per-project-root ProxyManagers for `scope: "project"` servers, with reference counting and 5-minute cleanup delay |
| **ConfigStore** | Watches and validates `config.json`, emits change events |
| **SkillManager** | Reads and searches skill files from disk |

### Desktop App (`apps/agent-manager/`)

A Tauri application with a React frontend and Rust backend:

- **React frontend** provides the GUI for managing skills, servers, and viewing status
- **Rust backend** handles file I/O (config, skills) and process status verification
- **Communicates via Tauri IPC** — the frontend calls typed Rust commands
- **Polls gateway status** by reading `status.json` every 5 seconds

### Shared Files on Disk (`~/.agent-manager/`)

The gateway and desktop app don't communicate directly. Instead, they share three types of files:

| File | Written by | Read by | Purpose |
|---|---|---|---|
| `config.json` | Desktop App | Gateway (watched) | Server configuration, settings |
| `skills/*.md` | Desktop App | Gateway | Skill content files |
| `status.json` | Gateway | Desktop App (polled) | Gateway runtime status (PID, port, URL, active sessions, server states) |

This file-based IPC keeps the two processes fully decoupled — either can run independently.
