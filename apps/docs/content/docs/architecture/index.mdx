## Component Responsibilities

### MCP Gateway Server (`apps/skill-mcp/`)

The Node.js/Bun process that runs as an MCP server:

- **Serves skills** from `~/.skill-management/skills/` via native tools
- **Proxies downstream servers** — connects to all enabled servers, namespaces their capabilities, and forwards requests
- **Watches configuration** — picks up changes to `config.json` without restart
- **Writes status** — periodically writes its PID and connection states to `status.json`

### Desktop App (`apps/skill-manager/`)

A Tauri application with a React frontend and Rust backend:

- **React frontend** provides the GUI for managing skills, servers, and viewing status
- **Rust backend** handles file I/O (config, skills) and process status verification
- **Communicates via Tauri IPC** — the frontend calls typed Rust commands
- **Polls gateway status** by reading `status.json` every 5 seconds

### Shared Files on Disk (`~/.skill-management/`)

The gateway and desktop app don't communicate directly. Instead, they share three types of files:

| File | Written by | Read by | Purpose |
|---|---|---|---|
| `config.json` | Desktop App | Gateway (watched) | Server configuration, settings |
| `skills/*.md` | Desktop App | Gateway | Skill content files |
| `status.json` | Gateway | Desktop App (polled) | Gateway runtime status |

This file-based IPC keeps the two processes fully decoupled — either can run independently.
