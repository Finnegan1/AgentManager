Agent Manager uses a **Turborepo** monorepo with **Bun** as the package manager. All applications and shared packages live in a single repository.

## Directory Layout

```
agent-manager/
├── apps/
│   ├── agent-manager/       # Tauri desktop app (React + Rust)
│   ├── mcp-gateway/           # MCP gateway server (Node.js/Bun)
│   └── docs/                # Documentation site (Fumadocs)
├── packages/
│   ├── shared-types/        # TypeScript type definitions
│   ├── ui/                  # Shared React UI components
│   ├── eslint-config/       # Shared ESLint configuration
│   └── typescript-config/   # Shared TypeScript configuration
├── package.json             # Root workspace configuration
├── turbo.json               # Turborepo task configuration
└── bun.lock                 # Dependency lockfile
```

## Applications

### `apps/agent-manager/`

The Tauri desktop application.

| Layer | Technology |
|---|---|
| Frontend | React 19, Vite 7, Tailwind CSS 4, shadcn/ui, CodeMirror 6 |
| Backend | Rust (Tauri 2.0) |
| IPC | Tauri command system |

The frontend lives in `src/` and the Rust backend in `src-tauri/`.

### `apps/mcp-gateway/`

The MCP gateway HTTP daemon.

| Aspect | Technology |
|---|---|
| Runtime | Node.js / Bun |
| MCP SDK | `@modelcontextprotocol/sdk` v1.12 |
| Client Transport | StreamableHTTPServerTransport (HTTP) |
| Downstream Transports | stdio, SSE, streamable-http |
| Config | Zod schema validation |

### `apps/docs/`

This documentation site.

| Aspect | Technology |
|---|---|
| Framework | Fumadocs on TanStack Start (SPA mode) |
| Content | MDX files with YAML frontmatter |
| Search | Orama full-text search |

## Shared Packages

### `packages/shared-types/`

TypeScript type definitions used across all applications:

- `Skill`, `SkillMetadata` — skill data types
- `AgentManagerConfig` — configuration structure
- `DownstreamServerConfig`, `TransportConfig`, `ServerScope` — server connection types
- `GatewayStatus` — runtime status types (PID, port, URL, active sessions, server states)

### `packages/eslint-config/` and `packages/typescript-config/`

Shared configuration for ESLint and TypeScript, ensuring consistent code style and compiler settings across all packages.

## Build System

Turborepo orchestrates builds with task dependencies defined in `turbo.json`. Key tasks:

- **`build`** — Compiles all applications and packages
- **`dev`** — Starts development servers
- **`lint`** — Runs ESLint across the workspace

Dependencies between packages are resolved automatically — for example, `apps/agent-manager` depends on `packages/shared-types`, so Turbo ensures types are built first.
