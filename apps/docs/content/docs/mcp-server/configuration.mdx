The gateway reads its configuration from `~/.skill-management/config.json`. This file is created automatically on first launch with sensible defaults.

## Config File Location

```
~/.skill-management/config.json
```

The desktop app reads and writes this file. The gateway watches it for changes and applies them automatically.

## Full Example

```json
{
  "version": "1.0",
  "gateway": {
    "autoStart": false,
    "port": 24842
  },
  "servers": {
    "database": {
      "name": "Database Tools",
      "enabled": true,
      "scope": "global",
      "transport": {
        "type": "stdio",
        "command": "bun",
        "args": ["/path/to/db-server/index.ts"],
        "env": {
          "DATABASE_URL": "postgresql://localhost:5432/mydb"
        }
      }
    },
    "web-search": {
      "name": "Web Search",
      "enabled": true,
      "scope": "global",
      "transport": {
        "type": "sse",
        "url": "https://api.example.com/mcp",
        "headers": {
          "Authorization": "Bearer sk-abc123"
        }
      }
    },
    "filesystem": {
      "name": "Filesystem Tools",
      "enabled": true,
      "scope": "project",
      "transport": {
        "type": "stdio",
        "command": "npx",
        "args": ["@mcp/filesystem", "."]
      }
    },
    "ai-helpers": {
      "name": "AI Helper Tools",
      "enabled": false,
      "transport": {
        "type": "streamable-http",
        "url": "https://ai.example.com/mcp"
      }
    }
  },
  "skills": {
    "directory": "~/.skill-management/skills"
  }
}
```

## Server Scope

Each downstream server has an optional `scope` field that controls how the gateway manages it:

| Scope | Behavior |
|---|---|
| `"global"` (default) | Started once when the gateway starts. Shared across all client sessions. |
| `"project"` | Started per project root. The `cwd` of stdio transports is overridden to the project directory. Sessions with the same project root share connections. |

See [Downstream Servers — Scoping](/docs/concepts/downstream-servers#scoping) for detailed examples.

## Client Configuration

To connect your AI client to the gateway, configure it to use the HTTP endpoint. For Claude Code, add to `.mcp.json`:

```json
{
  "mcpServers": {
    "skill-gateway": {
      "type": "http",
      "url": "http://127.0.0.1:24842/mcp"
    }
  }
}
```

<Callout>
  The gateway must be running before the client connects. Start it with `bun run dev:server` in the `apps/skill-mcp/` directory.
</Callout>

## Config Watching

The gateway uses `fs.watch()` to monitor the config file. Changes are:

1. **Debounced** at 200ms to prevent rapid-fire reloads
2. **Validated** against the Zod schema before being applied
3. **Diffed** against the previous config to determine what changed

Invalid config changes are logged and ignored — the gateway continues with the last valid configuration.

When a valid config change is detected, all active sessions are notified via `listChanged` notifications so clients can re-fetch the available tools, resources, and prompts.

## Default Configuration

When no config file exists, a default is created:

```json
{
  "version": "1.0",
  "gateway": {
    "autoStart": false
  },
  "servers": {},
  "skills": {
    "directory": "~/.skill-management/skills"
  }
}
```
