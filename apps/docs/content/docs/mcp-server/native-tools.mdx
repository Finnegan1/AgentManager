---
title: Native Tools
description: The 7 built-in MCP tools for skill management, marketplace, and gateway introspection.
---

The gateway exposes seven **native tools** that are always available, regardless of whether any downstream servers are connected.

## Skill Tools

### `skills__list`

Lists all available skills with their metadata.

**Parameters:** None

**Returns:** An array of skill metadata objects:

```json
[
  {
    "id": "typescript-best-practices",
    "name": "TypeScript Best Practices",
    "tags": ["typescript", "coding"],
    "description": "Guidelines for writing clean TypeScript.",
    "version": "1.0",
    "author": "Jane Doe",
    "created": "2026-01-15",
    "updated": "2026-03-20"
  }
]
```

### `skills__search`

Searches skills by a text query. The search is **case-insensitive** and matches against:
- Skill name
- Tags
- Description
- Full markdown content

**Parameters:**

| Name | Type | Description |
|---|---|---|
| `query` | `string` | The search query |

**Returns:** An array of matching skill metadata objects (same format as `skills__list`).

### `skills__get`

Retrieves the full content of a specific skill, including both metadata and markdown body.

**Parameters:**

| Name | Type | Description |
|---|---|---|
| `id` | `string` | The skill ID (directory name) |

**Returns:** The complete skill content including metadata, markdown body, and a list of additional files in the skill directory.

### `skills__sync`

Manually triggers a symlink sync from the skills directory to `~/.claude/skills/`. Useful after creating or deleting skills outside the desktop app.

**Parameters:** None

**Returns:** A confirmation message with the number of synced skills.

### `skills__marketplace_install`

Installs a skill from the [skills.sh](https://skills.sh/) marketplace. Accepts an install command copied from skills.sh, fetches the `SKILL.md` from GitHub, saves it locally, and triggers a symlink sync.

**Parameters:**

| Name | Type | Description |
|---|---|---|
| `command` | `string` | The install command from skills.sh, e.g. `"npx skills add https://github.com/owner/repo --skill skill-name"` |

**Returns:** A success message with the installed skill name, or an error if the skill was not found or already exists.

<Callout>
  The marketplace install fetches the `SKILL.md` directly from the GitHub repository's raw content. It tries multiple paths (`{skill}/SKILL.md` and `skills/{skill}/SKILL.md`) and branches (`main`, `master`) to locate the file.
</Callout>
```

## Gateway Tools

### `gateway__list_servers`

Lists all configured downstream servers with their current connection status.

**Parameters:** None

**Returns:** An object mapping server keys to their status:

```json
{
  "db": {
    "name": "Database Tools",
    "enabled": true,
    "connected": true,
    "toolCount": 5,
    "resourceCount": 2,
    "promptCount": 0
  },
  "ai": {
    "name": "AI Helpers",
    "enabled": true,
    "connected": false,
    "error": "Connection timeout"
  }
}
```

### `gateway__server_status`

Gets detailed status information for a specific downstream server.

**Parameters:**

| Name | Type | Description |
|---|---|---|
| `server_key` | `string` | The server key to query |

**Returns:** Detailed status including connection state, capability counts, and any error information.

<Callout>
  The `gateway__list_servers` and `gateway__server_status` tools report on **global-scope** downstream servers. Project-scoped server status is managed per session and per project root.
</Callout>
