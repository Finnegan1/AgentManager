The **MCP gateway server** is the core runtime component of Agent Manager. It's a Node.js/Bun HTTP daemon that implements the [Model Context Protocol](https://modelcontextprotocol.io/) and acts as a unified entry point for your AI clients.

## Running the Server

The gateway runs as an HTTP daemon on `127.0.0.1:24842`. Start it in development mode:

```bash
cd apps/mcp-gateway
bun run dev:server
```

Or for production (after building with `bun run build`):

```bash
cd apps/mcp-gateway
bun run start
```

You can specify a custom port with the `--port` flag:

```bash
bun run dev:server -- --port 3000
```

Then configure your AI client to connect. For Claude Code, add to `.mcp.json`:

```json
{
  "mcpServers": {
    "skill-gateway": {
      "type": "http",
      "url": "http://127.0.0.1:24842/mcp"
    }
  }
}
```

When started, the gateway:

1. Loads the configuration from `~/.agent-manager/config.json`
2. Starts watching the config file for changes
3. Initializes the global ProxyManager (connects to `scope: "global"` downstream servers)
4. Creates the ProjectConnectionPool for managing `scope: "project"` servers
5. Creates the SessionManager for handling multiple client sessions
6. Starts the HTTP server on the configured port
7. Begins writing status to `~/.agent-manager/status.json` every 5 seconds
8. Starts accepting MCP requests from AI clients via HTTP

## What It Exposes

The gateway exposes a unified set of capabilities to each connected AI client:

- **Native tools** — Built-in tools for skill management and gateway introspection
- **Proxied tools** — All tools from connected downstream servers, namespaced with the server key. Includes both global and project-scoped tools.
- **Proxied resources** — All resources from downstream servers, with namespaced URIs
- **Proxied prompts** — All prompts from downstream servers, namespaced

## Key Components

| Component | Responsibility |
|---|---|
| **HTTP Server** | Routes incoming HTTP requests to the correct session transport |
| **SessionManager** | Creates per-client `Server` + `Transport` pairs, manages session lifecycle, broadcasts notifications |
| **Global ProxyManager** | Orchestrates downstream connections for `scope: "global"` servers, shared across all sessions |
| **ProjectConnectionPool** | Manages per-project-root ProxyManagers for `scope: "project"` servers with reference counting |
| **ConfigStore** | Watches and validates `config.json`, emits change events |
| **SkillManager** | Reads and searches skill files from disk |
| **ToolRegistry** | Aggregates and namespaces tools from all sources |
| **ResourceRegistry** | Aggregates and namespaces resources |
| **PromptRegistry** | Aggregates and namespaces prompts |

## Graceful Shutdown

The gateway handles `SIGINT` and `SIGTERM` signals gracefully:

1. All client sessions are closed
2. All project connection pools are shut down
3. All global downstream connections are closed
4. The status file is cleaned up
5. The process exits cleanly
