---
title: Downstream Servers
description: External MCP servers that the gateway connects to and proxies.
---

A **downstream server** is any MCP-compatible server that the Skill Gateway connects to and proxies. Each downstream server can expose tools, resources, and prompts — all of which become available through the gateway under a namespaced prefix.

## Transport Types

Downstream servers can be connected using three different transport mechanisms:

### stdio

The most common transport for local MCP servers. The gateway launches a local process and communicates over stdin/stdout.

```json
{
  "name": "My Local Server",
  "enabled": true,
  "transport": {
    "type": "stdio",
    "command": "bun",
    "args": ["/path/to/server.ts"],
    "env": { "API_KEY": "sk-..." },
    "cwd": "/path/to/working/dir"
  }
}
```

| Field | Required | Description |
|---|---|---|
| `command` | Yes | The executable to run |
| `args` | No | Command-line arguments |
| `env` | No | Environment variables |
| `cwd` | No | Working directory |

### SSE (Server-Sent Events)

For remote MCP servers that use HTTP with Server-Sent Events for streaming.

```json
{
  "name": "Remote SSE Server",
  "enabled": true,
  "transport": {
    "type": "sse",
    "url": "https://example.com/mcp",
    "headers": { "Authorization": "Bearer token123" }
  }
}
```

### Streamable HTTP

For remote MCP servers using the newer streamable HTTP transport.

```json
{
  "name": "Remote HTTP Server",
  "enabled": true,
  "transport": {
    "type": "streamable-http",
    "url": "https://example.com/mcp",
    "headers": { "Authorization": "Bearer token123" }
  }
}
```

Both SSE and streamable HTTP support custom headers for authentication.

## Server Lifecycle

Each downstream server goes through these connection states:

```
connecting → connected → (error → reconnecting → connected)
                       → disconnected
```

- **connecting**: Initial connection attempt in progress
- **connected**: Successfully connected, capabilities discovered
- **error**: Connection failed; automatic reconnection begins (up to 10 retries with exponential backoff)
- **disconnected**: Intentionally disconnected (disabled or removed)

## Capability Discovery

When a downstream server connects successfully, the gateway discovers its capabilities:

1. **Tools**: All tools the server exposes
2. **Resources**: Static and template resources
3. **Prompts**: Pre-defined prompt templates

These are registered in the gateway's internal registries and become available to the AI client under the server's namespace prefix.

## Enabling and Disabling

Each server has an `enabled` flag. When a server is disabled:

- Its connection is closed
- Its tools, resources, and prompts are removed from the registries
- The server configuration is preserved for re-enabling later

Toggle this in the desktop app with the switch on each server card, or set `"enabled": false` directly in the config file.
